디어 오브제는 작가·소품샵·일반 사용자를 연결하는 온·오프라인 통합 플랫폼으로, 사용자는 지도 기반으로 소품샵을 탐색하고, 작가와 샵은 플랫폼 내에서 소통·입점 제안/승인·계약·정산까지 한 번에 처리할 수 있도록 설계된 원스톱 서비스입니다. 작가–소품샵 간 실시간 소통부터 입점 제안, 계약, 정산까지 이어지는 흐름을 한 곳에서 끊김 없이 제공해 사용자는 원하는 공간과 작품을 더 빠르게 찾고, 작가와 소품샵은 더 효율적으로 협업할 수 있도록 하는 것을 목표로 개발 중입니다.

담당 역할

  • 주문/결제/공지 도메인 REST API 설계·구현 및 공통 응답 래퍼(ApiResponse) 적용, 요청/응답 DTO로 계층 분리
  • 주문 생성·조회·상태변경·배송정보 등록·구매확정·취소 API(/api/v1/orders) 구현 및 주문번호(ORD-yyyymmdd-랜덤6자리) 생성 로직 적용
  • 주문 소유권 검증(validateOwner)으로 본인 주문만 접근 가능하도록 권한 제어, 주요 변경 기능(취소/상태/배송)에 일관 적용
  • 주문 통계 API(/api/v1/orders/stats) 구현: 일별 매출·인기상품 Top10·시간대별 주문 수를 Native Query + Projection으로 집계해 대시보드 데이터 제공
  • 토스페이먼츠 결제 연동 구현: 결제 생성/승인/취소/내역 조회 API(/api/v1/payments)와 TossPaymentsClient 연동, 주문상태·금액·주문번호 검증으로 결제 무결성을 확보하고, 사용자별 기간 조건 기반 결제 내역 조회를 페이지네이션으로 제공
  • 결제 Webhook 처리(/api/v1/payments/webhooks/toss) 구현: paymentKey 기반 결제 조회 후 내부 결제/주문 상태 동기화로 비동기 이벤트 반영
  • 공지사항 조회 API(/api/v1/notices) 구현: 카테고리 필터(공지/뉴스/이벤트/페스티벌) + 페이지네이션으로 최신순 목록 제공
  • 주문/결제/공지 도메인 REST API 설계·구현, ApiResponse 적용 및 DTO 계층 분리
  • 주문 생성/조회/상태변경 및 주문번호(ORD-yyyymmdd-랜덤6자리) 생성 로직 구현
  • 주문 소유권 검증(validateOwner) 기반 권한 제어 일관 적용
  • 주문 통계 API: Native Query + Projection으로 집계(일별 매출/Top10/시간대)
  • 토스페이먼츠 결제 승인·취소·내역 조회 및 Webhook 처리로 결제/주문 상태 동기화

하루새김

소개

https://github.com/Myeongseok-Kang/harusaegim

하루새김은 바쁜 일상 속에서도 일상의 사소한 순간과 감정을 빠르고 간편하게 남길 수 있는 Android 전용 감정일기 애플리케이션입니다. 사용자는 날짜, 위치, 활동, 감정의 네 가지 요소만 입력하면 되며, 서버에서는 OpenAI API를 활용해 자연스러운 1인칭 하루 일기를 자동으로 생성합니다. 누적된 기록을 바탕으로 7일·30일 감정 리포트를 제공하여, 사용자가 자신의 감정 흐름과 생활 리듬을 한눈에 돌아볼 수 있도록 설계했습니다.

담당 역할

  • GPT 기반 일기 생성 로직 구현 - 날짜·장소·한 일·기분을 입력받아 OpenAI API와 연동해 일기 생성, 장애 시 안전한 fallback 문장 제공
  • 감정 키워드를 기반으로 0~20 범위의 감정 점수를 계산하는 스코어링 함수와, 이를 활용한 7일·30일 감정 리포트 API(/api/reports/weekly, /api/reports/monthly) 구현
  • 일기 CRUD API 구현 및 사용자 소유권 검증(userId 일치 확인)으로 접근 제어 적용, 사용자+날짜 조합 중복 저장 방지로 데이터 무결성 강화
  • 토큰 기반 인증 방식 구현(로그인 시 토큰 발급/DB 저장/만료 관리) 및 인터셉터에서 토큰 검증으로 인증 필요한 API를 공통 관리
  • 회원가입/로그인/프로필 조회·수정 API 설계·구현, DTO/Validation으로 입력 검증 적용 및 응답 모델을 DTO로 분리
  • 예외 상황을 GlobalExceptionHandler로 정리해 검증 오류/비정상 요청/접근 불가 상황 등을 일관된 에러 응답(HTTP 상태코드 + 메시지)으로 반환하도록 구성
  • OpenAI API 연동으로 1인칭 일기 자동 생성 및 장애 대비 fallback 로직 구성
  • 감정 스코어링(0~20) + 주간/월간 리포트 API 구현
  • 일기 CRUD 및 소유권 검증, 사용자+날짜 중복 방지로 데이터 무결성 강화
  • 토큰 기반 인증(발급/만료/검증) 및 인증 필요한 API 공통 처리
  • DTO/Validation/GlobalExceptionHandler로 입력 검증과 예외 응답 체계화
